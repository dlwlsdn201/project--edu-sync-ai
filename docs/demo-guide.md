# EduSync AI — 데모 시연 가이드 (웹 · Vercel)

> 공모전·발표용. **브라우저(웹)** 기준으로 시연하는 절차입니다. 모바일 개발 빌드는 [`NEXT_TO_DO.md`](../NEXT_TO_DO.md)를 참고하세요.

---

## 1. 시연 전 체크리스트

| 항목 | 확인 |
|------|------|
| 배포 URL | Vercel 프로덕션(또는 프리뷰) 주소가 열리는지 |
| 환경 변수 | Vercel 프로젝트에 `EXPO_PUBLIC_SUPABASE_URL`, `EXPO_PUBLIC_SUPABASE_ANON_KEY`, `EXPO_PUBLIC_KAKAO_REST_API_KEY`, `EXPO_PUBLIC_GEMINI_API_KEY` 설정 |
| Supabase | `001_initial_schema.sql`, `002_role_nullable.sql` 적용, `kakao-auth` Edge Function 배포 및 Secrets |
| 카카오 | [카카오 Developers](https://developers.kakao.com) → Redirect URI: `https://<프로젝트>.supabase.co/functions/v1/kakao-auth` |
| 테스트 계정 | 카카오 로그인 가능한 계정 **2개**(교사 역할 / 학생 역할) 권장 |

**권장 환경**: 노트북에서 브라우저 창 **두 개**(또는 일반 창 + 시크릿 창) — 한쪽은 교사, 한쪽은 학생.

---

## 2. 판정 포인트 (30초 버전)

- **AI 퀴즈 생성**: 수업 자료 텍스트만으로 객관식 문제 자동 생성(Gemini).
- **실시간 대시보드**: 학생이 제출할 때마다 교사 화면 지표가 갱신(Supabase Realtime).
- **학습 리포트**: 학생 개인의 개념별 정답률·응시 이력.

---

## 3. 시연 스크립트 (약 5~7분)

### 준비 (30초)

1. 심사용으로 **배포 URL**을 즐겨찾기 또는 주소창에 준비합니다.
2. **창 A**: 교사용 카카오 계정으로 로그인할 준비.
3. **창 B**: 학생용 카카오 계정(또는 다른 브라우저 프로필).

---

### 3-1. 교사 — 로그인·역할·입장 코드 (1분)

1. **창 A**에서 사이트 접속 → **카카오로 시작하기**.
2. 최초라면 **역할 선택**에서 **교사** 선택.
3. **퀴즈 생성** 탭에서 상단 **입장 코드**를 확인합니다.
4. 심사위원에게: “이 코드로 학생이 수업에 들어옵니다.”

---

### 3-2. 교사 — AI 퀴즈 생성 (1~2분)

1. 퀴즈 **제목** 입력(예: `데모 수업 1차`).
2. **수업 자료**에 50자 이상 텍스트 입력(예: 교과 개념 요약).
3. **AI 퀴즈 생성** 클릭 → 로딩 문구가 나올 때까지 대기.
4. 완료 알림 후 **퀴즈 목록**으로 이동해, 새 퀴즈 카드가 보이는지 확인.

---

### 3-3. 학생 — 수업 참여·응시 (2~3분)

1. **창 B**에서 같은 배포 URL 접속 → 로그인 → **역할: 학생** (최초 선택 시).
2. **수업 참여** 탭에서 **창 A**에서 본 **입장 코드** 입력 → 참여.
3. **퀴즈** 탭에서 방금 만든 퀴즈 선택.
4. 한 문제를 **일부러 오답**으로 제출 → **힌트 보기** → AI 힌트 카드 노출 확인.
5. 나머지 문항 처리 후 결과 화면까지 진행(시간 부족 시 1~2문항만).

---

### 3-4. 교사 — 대시보드 (1분)

1. **창 A** → **대시보드** 탭.
2. 응시 인원·평균 정답률·학생 카드·개념 요약이 **새로고침 없이** 반영되는지 말로 설명(학생이 제출한 직후 확인).

---

### 3-5. 학생 — 리포트 (30초~1분)

1. **창 B** → **리포트** 탭.
2. 전체 정답률·개념별 정답률·응시 이력이 쌓이는지 짧게 보여줍니다.

---

### 마무리 한 줄 예시

> “교사는 자료만 넣으면 AI가 퀴즈를 만들고, 학생 응답은 실시간으로 보이며, 학생은 리포트로 취약 개념을 확인합니다.”

## 4. 시연 당일 트러블슈팅

| 증상 | 우선 확인 |
|------|-----------|
| 카카오 로그인 실패·KOE101 | Redirect URI, REST API 키, 카카오 로그인 활성화 — 상세는 [`NEXT_TO_DO.md`](../NEXT_TO_DO.md) 1절 |
| AI 퀴즈 생성 실패 | `EXPO_PUBLIC_GEMINI_API_KEY`, API 할당량·결제 상태 |
| 대시보드 숫자 안 바뀜 | Supabase Realtime·`student_logs` RLS, 학생이 **같은 수업·같은 퀴즈**에 응시했는지 |
| 학생에게 퀴즈 안 보임 | 학생이 **수업 참여**로 입장 코드 입력했는지, 교사가 **같은 수업**에 퀴즈를 만들었는지 |
